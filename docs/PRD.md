## 1. 产品愿景

### 1.1 愿景声明

**InsightForge** —— 锻造洞察，智绘未来。

打造一个 AI 驱动的技术报告生成系统，让深度分析和专业演示不再是专家的专属，而是每个需要表达的人都能掌握的能力。

### 1.2 核心价值

```
┌─────────────────────────────────────────────────────────┐
│                    InsightForge                         │
│                                                         │
│   📊 深度洞察 → 📝 专业报告 → 🎯 精彩演示              │
│                                                         │
│   • 方法论驱动：SWOT/PEST/波特五力等分析框架           │
│   • 知识增强：RAG 检索专业分析技巧和设计原则           │
│   • 双 Agent 协作：分析师 + 设计师无缝配合             │
│   • 一键输出：Markdown 报告 + PPT 结构设计             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3 目标用户

| 用户类型 | 痛点 | 使用场景 |
|---------|------|---------|
| 产品经理 | 没时间写分析报告 | 快速生成竞品分析、市场调研 |
| 技术顾问 | 报告质量参差不齐 | 标准化输出专业报告 |
| 创业者 | 缺乏分析方法论 | 商业计划书、路演 PPT |
| 研究人员 | PPT 设计是弱项 | 学术汇报、项目答辩 |
| 企业管理者 | 高效决策需要 | 战略分析、投资评估 |

## 3. 核心功能设计

### 3.1 功能清单

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| **输入** | 主题输入 | P0 | 支持自然语言描述 |
| | 框架选择 | P1 | 可选择特定分析框架 |
| | 资料上传 | P2 | 上传背景资料增强分析 |
| **分析** | 自动框架匹配 | P0 | 根据主题自动选择分析框架 |
| | 多框架组合 | P1 | 支持多个框架综合分析 |
| | 实时数据检索 | P2 | 接入外部数据源 |

| | PPT 结构设计 | P0 | JSON 格式的幻灯片结构 |
| | PPT 文件生成 | P1 | 直接生成 .pptx 文件 |
| | 演讲备注 | P2 | 每页幻灯片的演讲要点 |
| **知识库** | 分析框架库 | P0 | SWOT/PEST/波特五力等 |
| | PPT 技能库 | P0 | 设计原则/配色/图表 |
| | 模板库 | P1 | 报告和 PPT 模板 |
| | 行业知识库 | P2 | 垂直领域知识 |

## 5. 技术实现

### 5.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **Agent 框架** | CrewAI 1.10.1 | 多 Agent 协作编排 |
| **LLM** | OpenAI GPT-4 | 大语言模型（可替换） |
| **向量存储** | ChromaDB | 本地向量数据库 |
| **记忆后端** | Mem0 1.0.5 | Agent 长期记忆 |
| **PPT 生成** | python-pptx | Python PPT 库 |
| **运行环境** | Conda | Python 环境管理 |

## 7. 质量保证

### 7.1 输出质量标准

| 维度 | 标准 | 检验方法 |
|------|------|---------|
| **完整性** | 包含所有必要章节 | 自动检查模板字段 |
| **逻辑性** | 论证链条完整 | Agent 自检 |
| **准确性** | 分析框架应用正确 | 知识库验证 |
| **专业性** | 符合行业规范 | 专家审核（抽样） |
| **可读性** | 结构清晰、语言流畅 | 可读性评分 |

### 7.2 质量控制流程

```
输出内容
    │
    ▼
┌─────────────────────────────────────┐
│         自动质量检查                │
├─────────────────────────────────────┤
│ • 模板完整性检查                    │
│ • 格式规范性检查                    │
│ • 引用一致性检查                    │
└──────────────┬──────────────────────┘
               │
               ▼
        是否通过检查？
          /        \
        是          否
        │            │
        ▼            ▼
    输出交付    返回 Agent 修正
```

## 8. 成功指标

### 8.1 产品指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| **报告生成时间** | < 5 分钟 | 端到端时间 |
| **报告质量评分** | > 8/10 | 用户评分 |
| **PPT 可用率** | > 90% | 无需大改比例 |
| **知识检索命中率** | > 85% | 相关知识检索成功 |

### 8.2 用户价值

- ⏱️ **效率提升**：报告制作时间从 4 小时降到 5 分钟
- 📈 **质量提升**：结构化方法论，专业输出
- 🎯 **一致性**：标准化输出，质量稳定
- 📚 **知识沉淀**：方法论和最佳实践持续积累

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| LLM 幻觉 | 输出不准确 | 框架约束 + 事实核查 |
| 知识库质量 | 分析不专业 | 专家审核 + 持续更新 |
| 复杂主题 | 分析深度不足 | 分解主题 + 多轮对话 |
| API 成本 | 成本过高 | 本地模型 + 缓存优化 |
