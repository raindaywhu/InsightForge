## 1. 系统概述

### 1.1 系统目标

InsightForge 是一个 AI 驱动的技术报告生成系统，通过多个 Agent 协作，结合 RAG 知识增强，自动化产出高质量的技术报告和 PPT 演示。

### 1.2 核心能力

```
输入: 自然语言主题描述
处理: 多Agent协作 + RAG知识检索
输出: Markdown报告 + PPT文件
```

### 4.2 核心类设计

```python
# ┌─────────────────────────────────────────────────────────────────┐
# │                      TechReportCrew 类设计                       │
# └─────────────────────────────────────────────────────────────────┘

@CrewBase
class TechReportCrew:
    """
    技术报告生成 Crew
    
    职责:
    - 管理 Agent 和 Task 的定义
    - 加载和管理知识库
    - 协调 Agent 执行流程
    """
    
    # 配置文件路径
    agents_config: str = 'config/agents.yaml'
    tasks_config: str = 'config/tasks.yaml'
    
    # 知识库源
    knowledge_sources: List[StringKnowledgeSource]
    
    def __init__(self):
        """初始化: 加载知识库"""
        self.knowledge_sources = self._load_knowledge()
    
    def _load_knowledge(self) -> List[StringKnowledgeSource]:
        """
        加载知识库
        
        流程:
        1. 遍历 knowledge/ 目录
        2. 读取所有 .md 文件
        3. 创建 StringKnowledgeSource 实例
        4. 返回知识源列表
        """
        pass
    
    @agent
    def technical_analyst(self) -> Agent:
        """
        技术分析师 Agent
        
        职责:
        - 理解分析主题
        - 选择分析框架
        - 执行深度分析
        - 撰写技术报告
        """
        return Agent(
            config=self.agents_config['technical_analyst'],
            verbose=True,
            memory=True  # 启用记忆
        )
    
    @agent
    def presentation_designer(self) -> Agent:
        """
        演示设计师 Agent
        
        职责:
        - 解析报告结构
        - 设计信息架构
        - 创建幻灯片结构
        - 提供图表建议
        """
        return Agent(
            config=self.agents_config['presentation_designer'],
            verbose=True,
            memory=True
        )
    
    @task
    def analyze_task(self) -> Task:
        """
        分析任务
        
        输入: topic (主题)
        输出: Markdown 报告
        """
        return Task(config=self.tasks_config['analyze_task'])
    
    @task
    def design_task(self) -> Task:
        """
        设计任务
        
        输入: Markdown 报告 (来自 analyze_task)
        输出: JSON 结构
        """
        return Task(config=self.tasks_config['design_task'])
    
    @crew
    def crew(self) -> Crew:
        """
        组装 Crew
        
        配置:
        - agents: [technical_analyst, presentation_designer]
        - tasks: [analyze_task, design_task]
        - process: Process.sequential (顺序执行)
        - memory: True (启用 Crew 级别记忆)
        - knowledge_sources: 知识库列表
        """
        return Crew(
            agents=self.agents,
            tasks=self.tasks,
            process=Process.sequential,
            verbose=True,
            memory=True,
            knowledge_sources=self.knowledge_sources
        )
```

## 10. 性能优化

### 10.1 知识库优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| 向量缓存 | ChromaDB 持久化 | 避免重复向量化 |
| 检索优化 | 调整 top_k 参数 | 平衡准确性和速度 |
| 分块策略 | 合理切分文档 | 提高检索精度 |

### 10.2 LLM 优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| 模型选择 | gpt-4-turbo vs gpt-3.5 | 成本与质量平衡 |
| Token 限制 | 控制上下文长度 | 避免 token 超限 |
| 流式输出 | 使用 stream=True | 提升用户体验 |
